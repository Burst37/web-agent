## Next workstream: firecrawl-web integration

The user wants to integrate the agent experience into the main Firecrawl web app at `/Users/j/Documents/active-repos/Code/firecrawl-web`. Key points:

1. **BYOK (Bring Your Own Key)**: The `/v1/run` endpoint should accept model API keys in the request body, not just from env vars. Users select a model (Opus 4.6, GPT 5.4, Gemini 3.1) and enter their API key in the UI.

2. **Session-based keys**: API keys live in the browser session, not persisted server-side. The web app passes them with each request.

3. **Firecrawl API key passthrough**: For tool calls (search, scrape, interact), the user's own Firecrawl API key is used. Similar to how parallel agents work.

4. **Integration approach**: The firecrawl-web agent page would load the agent experience (similar to the Next.js template). The two apps share a design system, so it's mostly about dropping in the right API routes and components.

5. **For now**: Mock the API routes in the Next.js template. Don't worry about billing yet. Just demonstrate the BYOK pattern where API keys come through the request.
